# University Collaboration Risk Analysis Tool

Analyzes academic collaboration networks for the **University of Mississippi (ROR `02teq1165`)** using OpenAlex data. The tool surfaces *indirect* risk paths:

> **UM Author → Direct Collaborator → Flagged Second-Hop Collaborator**

The direct collaborator may not be risky, but their own collaboration network connects to authors affiliated with countries on a configurable review list. The dashboard is meant for university research / compliance teams to triage these paths.

## Architecture

```
OpenAlex API
    │  (ingest_openalex.py — cursor pagination, UM ROR filter)
    ▼
backend/data/authors.csv, works.csv, authorships.csv
    │  (precompute_graph.py — pairwise co-authorship edges)
    ▼
backend/data/collaboration_graph.json
    │  (FastAPI: /authors/um, /review/risk-paths, /review/summary, /review/shared-works)
    ▼
React + Vite + Cytoscape.js dashboard (frontend/)
```

The local CSV/JSON storage layer is isolated in `app/data_loader.py` so it can later be swapped for Postgres without touching the API or graph logic.

## Setup

### 1. Backend

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Pull UM works from OpenAlex into local CSVs
python app/ingest_openalex.py --max-pages 20 --per-page 100 --from-year 2018

# (Optional but recommended) Replace byline-style names ("A. Dye") with the
# canonical OpenAlex author names ("Andrew F. Dye"). Hits the /authors
# endpoint in batches of 50.
python app/enrich_authors.py

# Build the precomputed co-authorship graph
python app/precompute_graph.py

# Run the API
uvicorn app.main:app --reload
```

The API will be available at `http://localhost:8000`. Interactive docs at `http://localhost:8000/docs`.

Optional: set `OPENALEX_MAILTO=you@example.com` (or pass `--mailto`) so OpenAlex routes you through the polite pool.

### 2. Frontend

```bash
cd frontend
npm install
npm run dev
```

The dashboard will be available at `http://localhost:5173`.

## How to use the dashboard

1. The left sidebar loads UM authors from `/authors/um`. Search by name and click a name to select.
2. Click **Load Risk Paths**.
3. The graph shows only meaningful indirect risk paths:
   - **Red** = the seed UM author
   - **Blue** = direct collaborators that have at least one flagged second-hop
   - **Orange** = flagged second-hop collaborators (country in `review_countries.csv`)
4. Click any node to inspect details (institution, country, review reason, indirect risk count).
5. Click any edge to see the shared publications behind it.
6. Filters in the sidebar:
   - **Min edge weight** — hide weak collaborations (single shared paper, etc.).
   - **Country filter** — focus on a single review-country code (e.g. `CN`).
   - **Show direct collaborator labels** — off by default to keep the graph readable.

Direct collaborators with **no** flagged second-hop are intentionally not rendered. This is a risk-path explorer, not a generic co-authorship visualizer.

## Updating the review country list

Edit `backend/data/review_countries.csv`. The columns are:

| column      | meaning                                    |
|-------------|--------------------------------------------|
| `country`   | ISO 3166-1 alpha-2 code (e.g. `CN`, `IR`)  |
| `flag_reason` | Human-readable reason shown in the UI    |
| `risk_level`  | Free-text label (e.g. `High`, `Medium`)  |

After editing, either restart `uvicorn` or `POST /admin/reload` to refresh the cache.

## API reference

| Method | Path                               | Description                                                   |
|--------|------------------------------------|---------------------------------------------------------------|
| GET    | `/`                                | Health + graph metadata                                       |
| GET    | `/authors/um`                      | List UM-affiliated authors                                    |
| GET    | `/review/risk-paths?seed_author_id=` | Slice of the graph showing indirect risk paths              |
| GET    | `/review/summary?seed_author_id=`    | Aggregate counts + country breakdown                        |
| GET    | `/review/shared-works?source=&target=` | Shared publications behind an edge                       |
| POST   | `/admin/reload`                    | Clear in-memory caches after editing local data files         |

## Notes on terminology

The UI deliberately uses neutral language: **Needs Review**, **Review Match**, **Flagged Second-Hop**, **Indirect Risk Path**. It does not label individuals as "bad" or "problematic". The presence of an indirect risk path is a signal for human review, not a determination.
